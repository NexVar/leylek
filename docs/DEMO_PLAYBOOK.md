## 2. Pre-flight (1 minute before the jury opens the laptop)

```bash
cd ~/github/leylek
./scripts/e2e-demo.sh
```

This:
1. Re-seeds the demo dataset (idempotent; resets the AGGRESSIVE ad back to
   `active` so the optimizer can pause it again).
2. Drives a headless browser through the full flow.
3. Verifies `ads.status='paused'` on the AGGRESSIVE ad and that the newest
   `agent_logs` row is `optimizer/PAUSE_AD`.
4. Saves 5 screenshots into `e2e-out/` for paranoia.

A green run is your hands-off confidence. If anything fails the script
prints the API response so the issue is in front of you.

If you're cutting it tight, just rerun seed: `pnpm db:seed`.

## 4. What the jury is actually watching

- **Real Gemini call.** The reasoning text in the toast is generated by
  `gemini-2.5-flash` (PRD §16 fallback) with `responseSchema` enforced to
  match the `OptimizerDecision` Zod shape. The model picks the action and
  writes the Turkish prose; nothing in the toast is templated.
- **Real Cloudflare runtime.** 5 Workers + a per-campaign Durable Object +
  D1 SQL + KV — every box on the architecture diagram is a real binding,
  not a mock. The `/api/health` aggregate probes each upstream via Service
  Bindings, no internet egress between services.
- **Sim ad-platform.** The only piece behind a `LEYLEK_AD_PLATFORM=sim`
  flag is the outbound call to Google Ads. `workers/publisher-agent/src/clients/real-google-ads.ts`
  ships the real REST integration today — flip the env var when the dev
  token Standard access lands (PRD §10 / §17).

## 5. Defensive lines (if the jury probes)

- **"Why simulator?"** — _"Production Google Ads code is in the repo —
  open `workers/publisher-agent/src/clients/real-google-ads.ts`. Token
  Standard access is a 1-2 week external approval that doesn't fit the
  hackathon. Demo uses the sim adapter; flipping `LEYLEK_AD_PLATFORM=real`
  is one line."_
- **"How is this not just an LLM wrapper?"** — _"Three distinct agents in
  separate Workers — content-agent, optimizer-agent, publisher-agent —
  each with single-responsibility. The optimizer holds per-campaign atomic
  state in a Durable Object; race-conditions are impossible by design."_
- **"What if Gemini is slow / down?"** — _"Both Pro and Flash quotas
  configured. Optimizer retries once on schema-parse failure, then writes
  an `OPTIMIZER_FAILED` agent_log so the dashboard sees the outage rather
  than silently breaking."_
- **"Where's the user-facing safety?"** — _"Reasoning is the product. Every
  agent decision lands in `agent_logs` with the Gemini-generated Turkish
  rationale + confidence score. The user can always ask 'why did you do
  that?' and get a literal answer."_

## 6. Recovery

| Symptom                                                  | Recovery                                                                       |
|----------------------------------------------------------|--------------------------------------------------------------------------------|
| AGGRESSIVE ad already paused (a previous demo ran)        | `pnpm db:seed` — instant reset, idempotent                                     |
| Toast not appearing                                       | Check Gemini quota (`/api/health` shows `optimizer.status`)                    |
| Cookie not sticking after dev-login                       | Browser blocked third-party cookies? Open in incognito + don't block 3rd-party |
| Pages site says "X-Frame-Options not allowed"             | You hit the preview URL; use https://leylek-web.pages.dev                      |
| `pnpm db:seed` errors with "ids don't match"              | Set `LEYLEK_SEED_FORCE=1`                                                      |
| Gateway 502 on `/optimize-now`                            | Gemini quota — wait 60 s, retry; cooldown is auto-handled by retry-once        |
